# Resource Bank Style Guide

## Status
This file is the binding visual baseline for homepage redesign work.
If the current implementation differs from this guide, follow this guide.

## Design intent
The site should feel like:
- a branded study platform
- a designed visual identity, not generic ed-tech UI
- paper-cut / poster / card-based composition
- clear hierarchy, large visual blocks, and memorable shapes

The homepage is a brand page first.
The library and viewer are product pages second.

## Primary visual direction
Light theme:
- J2 direction: warm earth-tone paper UI
- bold outlines
- flat color blocking
- playful but structured composition
- card shapes should feel placed, not generated by a default UI kit

Dark theme:
- J2-dark direction: midnight paper-cut composition
- strong contrast
- luminous accent colors
- same structural language as light mode, not a different product

## Must
- Use strong visual hierarchy with large blocks and clear focal areas.
- Use bold outline language for homepage cards, buttons, labels, and hero surfaces.
- Keep homepage composition poster-like, not dashboard-like.
- Keep library and viewer calmer than homepage.
- Distinguish document resources from interactive resources visually.
- Reuse one branded language across homepage, library, and viewer.
- Prefer custom-shaped surfaces, stickers, plaques, and tags over generic SaaS cards.

## Prefer
- Slight tilt or asymmetry on homepage cards where it helps composition.
- Big hero surface with layered paper shapes.
- Visually distinct tags and chips with strong silhouette.
- Decorative shapes that feel intentional and sparse.
- Warm paper-like backgrounds in light theme.
- Neon-edge or glow-accent dark theme, but only where it supports hierarchy.

## Avoid
- Generic rounded SaaS cards
- default-looking pill buttons everywhere
- pale decorative inner borders that do not contribute to hierarchy
- glassmorphism
- thin, overly delicate ornament as the main identity
- homepage layouts that look like a restyled library page
- visually dense homepage sections with too many equally weighted items

## Homepage composition rules
Homepage should include:
- strong branded header
- large hero
- primary CTA group
- search entry
- feature/category cards
- featured documents
- a clearly separate interactive section

Homepage should not:
- feel like a raw listing page
- expose too much dense filtering UI above the fold
- give equal weight to every content block

## Library rules
Library should:
- inherit the brand language
- be calmer and more scan-friendly
- preserve usability over decoration
- use branded cards, tags, filters, and headings

Library should not:
- feel like the homepage poster duplicated
- overuse tilt, strong decorative clutter, or oversized stickers

## Viewer rules
Viewer should:
- feel part of the same family
- be the calmest of the three
- prioritize reading comfort
- keep branded shell, tags, metadata, and return controls

Viewer should not:
- become visually louder than the document content
- use decorative layout that competes with reading

## Tokens to preserve
- outline-driven visual language
- warm paper palette in light mode
- strong contrast in dark mode
- branded chip/tag/button system
- visually distinct resource-card system
- separate interactive-entry treatment

## Implementation priority
When trade-offs appear, prioritize in this order:
1. composition
2. hierarchy
3. card/button/tag language
4. decorative details
